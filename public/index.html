URL Shortener Microservice

User stories:

  1. I can pass a /newlink/ and a URL as a parameter and I will receive a shortened URL in the JSON response.
  2. If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the response will contain an error instead.
  3. When I visit that shortened URL, it will redirect me to my original link.

Example Creation Usage

https://dry-dusk-96175.herokuapp.com/newlink/https://www.google.com
https://dry-dusk-96175.herokuapp.com/newlink/http://foo.com:80

Example Creation Output

{ "original_url":"http://foo.com:80", "short_url":"my-herokuapp.com/8170" }

Usage

https://dry-dusk-96175.herokuapp.com/apple2871
